# 気分可視化日記アプリ

### 🌟 サービス概要
日々の「気分」をデータとして記録し、自分の感情の波を可視化することで、どのような条件下で気分が変化するのかを分析できる日記アプリです。

### 🚀 制作背景（なぜこれを作ったか）
私自身、日々の気分の変化が何に起因しているのかを客観的に把握したいと考えたことがきっかけです。「気分の上がり下がり」を可視化することで、自身のメンタルケアや生活習慣の改善に役立てるツールを目指して開発しました。

### 🛠️ 技術的なこだわりと解決した課題
**「仕組みを理解してエラーを解決する力」**
開発中、`posts_controller.rb` のストロングパラメーター周辺でエラーに直面しました。
単にエラーメッセージを検索するだけでなく、**「パラメーターがどこから来て、どこで定義されているのか」**というデータの流れ（MVCの仕組み）をソースコードレベルで追跡しました。その結果、Railsのセキュリティ構造を深く理解することができ、自力での解決に繋げることができました。

### 🌈 目指すエンジニア像
私は、**「ユーザーの『欲しい』を具現化し、代替案まで提案できるエンジニア」**を目指しています。
単に指示通りに実装するだけでなく、ユーザーの視点に立ち、技術的な制約がある場合でも「これなら実現可能です」という代替案を論理的に提示できるような、ビジネスに貢献できる技術者でありたいと考えています。

## Docker開発環境での起動方法

このリポジトリは、Docker Compose（v2以降の `docker compose` コマンド）で開発環境を構築できます。

### 前提
- Docker Desktopがインストールされていること

### 起動手順
1. イメージのビルドとサービス起動

   ```sh
   docker compose up --build
   ```

2. 別ターミナルでDB作成・マイグレーション

   ```sh
   docker compose run web bin/rails db:create db:migrate
   ```

3. サービスの停止

   ```sh
   docker compose down
   ```

### 備考
- DBの接続情報は `config/database.yml` でDocker用に自動設定されます。
- `docker-compose` ではなく `docker compose` コマンドを使用してください。

### 2回目以降の起動
2回目以降はビルド不要なので、以下のコマンドで素早くコンテナを立ち上げられます。

```sh
# バックグラウンドで起動
docker compose up -d

# フォアグラウンドで起動（ログを表示）
docker compose up
```
